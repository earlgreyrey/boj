<h3 id="emails">1. Emails</h3>
<p><strong>배경</strong></p>
<p>실무자는 Runtime Analysis 기능을 사용하기 위해 제품을 구입했다.</p>
<p><strong>HBGary</strong></p>
<p>악성코드를 탐지, 격리 및 분석할 수 있는 다양한 악성코드 분석 도구와 컴퓨터 포렌식과 관련한 분석 도구를 제공 하는 회사다.</p>
<p><strong>문제</strong></p>
<p>Active Reversing 기능 중 Analyze Binary 수행 후 Global 섹션에 접근 할 수 없는 버그가 발생했다.</p>
<p><strong>실무자의 문제 제기</strong></p>
<p>버그 발생 신고 메일을 HBGary에 보낸 후 증거로 버그가 발생하는 영상과 현재 사용 중인 버전의 사용 설명서를 보냈다. 하지만 한달 간 답장을 받지 못해, 다시 문의하였다.</p>
<p><strong>HBGary의 답변</strong></p>
<p>Responder 디버거는 아래의 이유로 지원이 종료될 예정이라고 말했다.</p>
<ul>
<li>
<p>우수한 무료 디버거가 존재하기 때문에 디버거 개발에 투자하는 것은 의미가 없다고 생각했다.</p>
</li>
<li>
<p>새로 개발중인 “Flypaper”는 디버거보다 사용하기 편하고, 소프트웨어 실행과 관련된 런타임 정보들을 자동으로 수집한다.</p>
</li>
</ul>
<p>새로 출시될 “Flypaper” 라는 동적 분석 디버거로 대체될 것이므로 “Flypaper”를 사용하고 피드백을 요청했다.</p>
<p><strong>실무자의 반응</strong></p>
<ul>
<li>
<p>버그에 대한 답변 대신 새로운 대안 제시에 실망했다.</p>
</li>
<li>
<p>실무자는 HBGary가 버그 패치에 관심을 가지길 바랐다.</p>
</li>
<li>
<p>Runtime Analysis가 동작하지 않는 정확한 이유와 Responder를 사용하는 다른 Client들도 같은 문제가 발생했는지 알려달라고 요구했다.</p>
</li>
</ul>
<p><strong>HBGary 내부 반응</strong></p>
<ul>
<li>
<p>버그 패치 대신 향후 출시될 “Flypaper” 선사용을 보상으로 제공하고, “Flypaper”에 대한 피드백을 받고자 했다.</p>
</li>
<li>
<p>실무자의 반응에 당황하며 버그 패치나 환불을 논의했다.</p>
</li>
<li>
<p>후에 제품 구매의사가 있다면 라이선스 갱신 할인을 고객에게 제공하고자 했다.</p>
</li>
</ul>

<h3 id="기술적으로-어떤-것이-문제인가">2. 기술적으로 어떤 것이 문제인가?</h3>
<p>다음은 동적 분석(Active Reversing)을 사용하기 위해 타겟 프로세스에 “Analyze Binary”를 수행하는 과정이다.</p>
<ol>
<li>
<p>Responder와 “WintelNodeAgent” 간 세션을 연결한다.</p>
</li>
<li>
<p>타겟 프로세스를 실행한다.</p>
</li>
<li>
<p>WintelNodeAgent에 attach할 타겟 프로세스를 선택한다.</p>
</li>
<li>
<p>WintelNodeAgent가 실행되고 있는 머신의 모든 프로세스와 DLL 목록을 가져온다.</p>
</li>
</ol>
<p><img src="https://lh3.googleusercontent.com/GNDMsrpPytfdIIoiY1VYZmsC0b7ryIKKVXTwnGfbmlrsjrV5wFehu06NNOmL7dDw-ytuLZaIK9g90njJLmmMuq036J7CktxuySVPlaCqvMYwv-zRIYdNBBFYm4fr4DZnyDwv1bM_" alt=""></p>
<p>&lt;attached 프로세스 정보&gt;</p>
<p>프로세스와 DLL 목록에서 원하는 바이너리를 선택하면 Responder가 설치된 머신으로 가져올 수 있다. 이는 메모리에서 직접 추출한 바이너리이기 때문에 Packer 등 보호 기법이 해제된 상태다.[5]</p>
<p><img src="https://lh5.googleusercontent.com/_cO0ZorhKhand2sSZVMg4_IYgRIKlKoKrCm5cQPspWvL6zK-ejn0gX3_R0xAlNPUd1OA5HNCo3cyhOp0SJpn-1sWKPgqD_wyXB4ahx40wODhALM-_o400POGUYkCZXF_6rK9X8co" alt=""></p>
<p>&lt;정상적으로 Global 섹션이 생긴 모습&gt;</p>
<p>가이드에서 타겟 프로세스에 "Analyze Binary"를 수행하면 "Global"이라는 섹션이 활성화 되며 sample points를 추가할 수 있게 된다.[1]</p>
<p><img src="https://lh5.googleusercontent.com/lXp4vkl0Gp9HUKKIfzbA99xTeQbP8u-RhUxiXc2npLAyjjCLP8ob_6Kt1zIrYyn-5LbAKpTIUeK5agQSNhvKlcLOsWMpD0gC1fo_pFjFe1190GSeFInfDuF2PDcbf0llDcKcPrqQ" alt=""></p>
<p>&lt;실무자의 Responder에서는 Global 섹션이 없는 모습&gt;</p>
<p>그러나 HBGary에서 제공한 Responder에서는 이 섹션이 활성화 되지 않아 동적 분석이 불가능하다.[2]</p>
<p>유출된 회사 내부자 간의 메일에 의하면 BlackHat에서 시연한 내용의 대부분은 연구 결과로써 일부 기능만 Responder에 구현이 되었다. 그러나, 아래 세가지 기능은 제공하지 않았다.<strong>[3]</strong></p>
<ul>
<li>
<p>live debugging</p>
</li>
<li>
<p>class reconstruction</p>
</li>
<li>
<p>phase-space analysis</p>
</li>
</ul>

<h3 id="실무자가-원했던-기능은-어떤-것인가">3.실무자가 원했던 기능은 어떤 것인가?</h3>
<p>HBGary Responder는 윈도우 운영체제에서 활성 메모리 전체 내용을 추출하고, 분석하여 컴퓨터 조사에 필요한 운영체제와 프로세스 정보를 찾아낸다. 이 때 수집한 내용은 커널과 사용자 모드에서의 오브젝트, 구조체, 바이너리, 그 외 유용한 아티팩트들이다.<strong>[5]</strong></p>
<p>HBGary Responder의 커널 디버거, 가상 머신 메모리 분석, 메모리 덤프는 모두 좋은 기능이지만 실무자가 요구한 것은 아니었다. 실무자는 “Runtime Analysis” 기능을 원했다.[4]</p>
<p>Runtime Analysis 기능 중 하나인 Active Reversing은 타겟 프로세스의 행위(파일 열기, 읽기, 쓰기 등)를 IDA 같이 실행 순서에 따라 그래프 구조로 표현한다. 또 다른 기능인 Tracing은 원하는 API 콜이나 현 브레이크 포인트 위치에서 몇번째 함수 호출 때 브레이크 포인트를 거는 작업을 할 수 있다. 호출 뿐만 아니라 모든 인스트럭션을 잡을 수 있고, 데이터를 볼 수 있다.<strong>[6]</strong></p>

<h3 id="hbgary의-문제점은-무엇인가">4. HBGary의 문제점은 무엇인가?</h3>
<ol>
<li>책임감 결여</li>
</ol>
<ul>
<li>
<p>실무자의 문의에 즉각 응답은 커녕 1달이 지나도 답장을 주지 않았다.</p>
</li>
<li>
<p>시장에 우수하고 무료인 디버거가 있기 때문에 디버거 개발은 의미가 없다고 하였다.</p>
</li>
<li>
<p>받은 문의에 대해 다른 담당자에게 전달만 하고 이후 진행사항을 체크하지 않았다.</p>
</li>
<li>
<p>자사의 디버거 기능을 모르는 책임자가 실무자에게 답장을 보냈다.</p>
</li>
<li>
<p>제대로 구현되지 않은 기능의 허위 광고로 프로그램 구매 유도했다.</p>
</li>
</ul>
<ol start="2">
<li>사후대책 미흡</li>
</ol>
<ul>
<li>
<p>실무자에게 디버거를 더 이상 지원하지 않을 것이라 했다.</p>
</li>
<li>
<p>버그를 패치하지 않았다.</p>
</li>
</ul>
<ol start="3">
<li>고객의 pain point 해결 실패</li>
</ol>
<ul>
<li>
<p>실무자가 원하는 답을 주지 못했다.</p>
</li>
<li>
<p>오히려 아직 릴리즈 하지 않은 “Flypaper” 설명문서와 다운로드 링크를 보내며 피드백을 받고자 했다.</p>
</li>
<li>
<p>실무자가 원하는 Runtime Analysis 기능이 작동하지 않았다.</p>
</li>
</ul>
